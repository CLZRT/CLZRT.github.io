IAM

概念

AWS Identity and Access Management (IAM) 是一项 Web 服务，可让 Amazon Web Services (AWS) 客户在 AWS 中管理用户和用户权限。借助 IAM，您可以集中管理用户、访问密钥等安全凭证，以及用于控制用户可以访问哪些 AWS 资源的权限

功能

  • 管理对AWS资源访问
    • 控制谁可以终止Amazon EC2实例
  • 定义精细的访问权限(假如使用密码,密钥本地登录.权力大小难管理)
    • 谁可以访问资源
    • 可以访问哪些资源, 以及用户可以对资源执行哪些操作
    • 如何访问资源

IAM概念理解

用户

  • 可以使用AWS账户进行身份验证的人员, 或者应用程序

例子: 我在班级舞台剧中表演一棵树,其中我是用户,班级舞台剧是活动,树是角色.我只有在班级舞台剧上才能是树

或者

班助与普通大学生,特定事件.

用户组

  • 雨课堂里可以看课堂界面的人,只要被加到学生组就可以了

角色

  • 为发出AWS服务请求授予一组权限的有用机制(套皮),
  • 跟用户组内的用户有什么区别?
    • 某件事需要多个用户组权限,创造角色,给用户套上.事情完成后取消这个角色.

权限

  • 谁(who)在什么时候(when)可以在哪里(where)怎么(how)做什么事情(what)
    • who:角色(套皮)
    • when:权限是否过期
    • where:登陆设备
    • how:通过哪个端口(哪种AWS服务实现)
    • what:更改数据
