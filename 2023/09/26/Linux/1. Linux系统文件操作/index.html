Linux Filesystem

Created2023-09-26|Updated2023-09-26
|Post View:

文件系统目录结构

  • /home: 用于存放各个普通用户的文件

  • /usr(Unix System Resources): 用于存放 Unix 系统的资源文件

下载第三方软件在这

  • /etc (/etcetera) :本来是用来放计算机一直(常)要用的 config/data/socket 文件的, [本来在boot/dev 中的诸如此类的文件] ,现在他作为 LInux machine 的大脑所在

配置Linux系统在这

  • /var (/varible) :保存系统的缓存数据

看日志,在这看 (观察进程执行情况在这)

文件类型,权限

目录文件操作

创建/删除/更换 目录

cd : change directory

pwd: print working directory

mkdir: make directory

rmdir: remove directory(删除空目录)

rm: remove(删除目录及其文件)

查看目录里的文件

ls

-a :所有文件

-A: 所有文件 - (“.” +”..”)

-F: 对文件进行分类 (可执行文件/目录文件/socket文件/管道文件)

-i : 列出inode位置,不列出文件属性

-l : 列出文件属性等信息

-R: 子目录也列出来

普通文件操作

index node (innode)

意义: 保存文件的相关信息(属性,大小,访问权限)

文件名保存于文件目录中,一个文件对应一个索引,对应多个文件名

创建新文件名(同一文件)

  • ln 原始文件名 (新)硬链接文件名(指向inode 节点)
  • cp -l 原始文件名 (新)硬链接文件名

复制/移动/删除文件

  • cp :copy
  • rm: remove
  • mv: move
  • ln:link (用于建立硬链接与软(符号)链接,硬链接是指向同一索引节点的另一文件名,符号链接(软)是指向硬链接的文本文件)

文件内容查询

  • cat
  • tac
  • head
  • tail
  • more
  • less

创建/编辑文件

  • gedit :创建/编辑文本文件(有字符编码)

  • touch: 创建空文件

  • dd: 创建指定大小,内容初始化为0的文件

文件/目录文件 属性修改

chmod 命令

通配符

*: 匹配多个字符

?: 匹配单个字符

文件压缩与打包

tar

步骤:

  1. 压缩
  2. 打包

Linux命令格式

  • 命令手册查询
    • man pwd
    • info pwd

输入输出重定向 和 管道

  • 输入重定向
    • 命令 < 文件名
  • 输出重定向
    • 命令 > 文件名
  • 管道
    • 命令1 | 命令2 | 命令3
  • 命令替换
    • 命令1 ‘命令2’
