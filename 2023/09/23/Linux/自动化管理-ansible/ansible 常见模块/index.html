ansible 模块

tips

文件管理模块

file 模块

  1. 指定路径

    1. path
    2. name

  2. 指定行为

    1. touch： 创建新文件
    2. directory： 创建新目录
    3. file： 修改文件
    4. absent：删除问件
    5. link: 创建软连接
    6. hard： 创建硬链接

  3. 其他参数

    1. owner：指定所有者
    2. group：指定所属组
    3. mode：指定权限
    4. setype：指定上下文

  4. example

copy 和fetch 模块

copy 从本机复制到目标机器

fetch 从目标机器复制到 本机

  1. copy
    1. ansible server2 -m copy -a “src=/etc/ansible/hosts mode=000 owner=tom dest=/opt”
    2. ansible server2 -m copy -a “content=’123’ dest=/opt/11.txt”
  2. fetch
    1. ansible server2 -m fetch -a “src=/opt/hosts dest=.”
    2. 创建与远端主机同名目录，表示从哪里来的

软件包管理模块

yum_repository 模块（设置 yum源）

yum模块

  1. 参数说明
    1. name：软件包名称
    2. state：指定yum行为
      1. present、installed：下载，安装软件包
      2. absent，removed: 卸载软件包
      3. latest：更新

服务管理模块

service 模块

参数

  1. name：指定哪个服务
  2. enabled： 设置服务自动启动
  3. state：服务行为（started，stopped restarted）

磁盘管理模块

parted模块 （分区）

前提条件

目标机内有未分区的硬盘

参数

  1. device
  2. number
  3. part_start
  4. part_end
  5. state: present/absent

例子

filesystem模块 （格式化/文件系统格式）

前提条件

分区已经创建好

参数

  1. dev
  2. fstype（文件系统类型）
    1. xfs
    2. nfs
  3. force(默认为no 是否强制格式化)

例子

mount 模块（挂载文件系统）

前提

已经 文件系统格式化分区

参数

  1. src：挂载设备
  2. path：挂载点
  3. fstype：挂载的文件系统类型
  4. opts：挂载选项（默认为defaults）
  5. state：挂载行为
    1. mounted：挂载，再写入/etc/fstab
    2. present:写入 /etc/fstab,但并未挂载
    3. unmounted：当前卸载，但并未把记录从 /etc/fstab 删除
    4. absent：卸载，并从/etc/fstab 中删除

例子

lvg 模块（管理卷组）

参数

  1. pvs：指定物理卷
  2. vg：指定卷组名称
  3. pesize：指定PE的大小
  4. state
    1. present：创建卷组
    2. absent：删除卷组

lvol 模块 （管理逻辑卷）

前提

创建好卷组，（大房间）

没有卷组 无法创建逻辑卷（大房间里的各个隔断间）

参数

  1. vg：用于指定在那个卷组上划分逻辑卷
  2. lv：用于指定逻辑卷名称
  3. size：用于指定逻辑卷大小
  4. state
    1. present：创建逻辑卷
    2. absent：删除逻辑卷

firewalld 模块

参数

  1. service：开放哪个服务
  2. port：开放那个端口 （port=80/tcp)
  3. permanent=yes 设置永久生效，不存在默认值
  4. immediate=yes 设置当前生效，默认为不
  5. state
    1. enabled：用于创建规则
    2. disabled：用于删除规则
